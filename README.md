# LLM‑ассистированный перевод (МКК переводчиков) — GitHub Pages

Это статический сайт‑ресурс (цифровой продукт) для организации учебного модуля:
**кейс → промпты A/B/C → аудит генерации → постредактирование + комментарий → оценивание по ПА/РЦИ/КС**.

## 1) Быстрый запуск сайта (GitHub Pages)
1. Создайте репозиторий на GitHub (например: `llm-icc-module`) и загрузите в корень файлы сайта.
2. В репозитории: **Settings → Pages**  
   Source: *Deploy from a branch* → Branch: `main` → Folder: `/(root)` → Save
3. Откройте ссылку: `https://<username>.github.io/<repo>/`

## 2) Описание проекта (кратко)
Ресурс разработан как электронный учебно‑методический продукт по итогам ВКР.
Перевод и генерация выполняются **во внешней LLM** (ChatGPT/Qwen/DeepSeek и т.п.), а сайт:
- фиксирует параметры кейса (адресат, жанр, цель, стратегия, исходный текст)
- генерирует промпты A/B/C для сопоставления вариантов
- собирает и сохраняет 3 результата генерации
- предоставляет интерактивный аудит‑лист и экспорт CSV
- помогает оформить финальный перевод и переводческий комментарий (отчёт TXT)
- считает баллы по ПА/РЦИ/КС и формирует файл `assessment.txt`

## 3) Как пользоваться (workflow)
1. **Главная**: заполните кейс → *Сохранить кейс*.
2. **Этап 1**: нажмите *Сгенерировать промпты A/B/C* → отправьте промпты в LLM → вставьте 3 результата → выберите лучший → *Сохранить Этап 1*.
3. **Этап 2**: заполните аудит‑лист → *Сохранить аудит* → при необходимости *Экспорт CSV*.
4. **Этап 3**: вставьте финальный перевод → *Сгенерировать комментарий из аудита* → *Скачать отчёт (TXT)*.
5. **Оценивание**: выставьте ПА/РЦИ/КС → скачайте `assessment.txt`.

## 4) Техническая документация (что обновляем и как)
### Структура репозитория
- `index.html` — форма кейса + краткие инструкции
- `stage1.html` — промпты A/B/C и поля для вставки результатов генерации
- `stage2.html` — аудит‑лист (таблица) + экспорт CSV
- `stage3.html` — финальный перевод + комментарий + отчёт TXT
- `assessment.html` — калькулятор баллов ПА/РЦИ/КС + `assessment.txt`
- `downloads/` — DOCX/PDF шаблоны и примеры кейсов
- `assets/styles.css` — стили
- `assets/script.js` — логика страниц + сохранение данных (localStorage)

### Как добавлять/обновлять материалы
- Добавьте новые файлы в `downloads/` (DOCX/PDF/TXT).
- Обновите кнопки/ссылки в `downloads.html` (если меняете имена файлов).

### Как обновлять тексты и инструкции
- Тексты страниц — внутри соответствующих `.html`.
- Общие стили — в `assets/styles.css`.

### Версионирование (рекомендуется)
- Для каждого изменения делайте commit с коротким сообщением:
  - `Add case 2 (humour)`
  - `Improve audit list`
  - `Update assessment rubric`
- При крупных изменениях добавляйте запись в раздел **Changelog** ниже.
