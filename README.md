# LLM-ассистированный перевод — автономный учебно-методический сайт

**Версия:** 5.0  
**Статус:** демонстрационная версия продукта магистерской работы  
**Формат:** статический сайт для GitHub Pages  
**Основной режим работы:** полностью автономный, без обязательного API и без платных ключей

## 1. Назначение проекта

Сайт является цифровым учебно-методическим ресурсом для организации работы студентов-переводчиков с LLM-ассистированным переводом.

Ресурс поддерживает последовательную учебную процедуру:

1. ввод переводческого кейса;
2. генерация и редактирование промптов A/B/C;
3. получение вариантов перевода во внешней LLM вручную;
4. аудит результата;
5. постредактирование;
6. составление переводческого комментария;
7. формирование мини-глоссария;
8. оценивание по критериям ПА / РЦИ / ПК;
9. экспорт отчётов и материалов.

## 2. Основной режим без API

По умолчанию сайт работает без серверной части и без API-ключей.

В файле `assets/config.js` установлено:

```js
window.LLM_API_BASE = "";
```

Это означает, что студент работает по бесплатному автономному сценарию:

1. заполняет кейс на главной странице;
2. переходит на Этап 1;
3. генерирует промпты A/B/C;
4. копирует выбранный промпт во внешнюю LLM;
5. вставляет полученный ответ обратно на сайт;
6. продолжает аудит, постредактирование и оценивание.

Сайт не требует OpenAI API, Anthropic API или другого платного сервиса.

## 3. Опциональный режим API

Папка `worker/` сохранена как дополнительная возможность. Она не обязательна для демонстрации и работы сайта.

Если позже понадобится встроенная генерация или AI-оценка через серверную прослойку, можно развернуть Cloudflare Worker и прописать его адрес в `assets/config.js`:

```js
window.LLM_API_BASE = "https://<worker-name>.<subdomain>.workers.dev";
```

API-ключ нельзя хранить в репозитории, HTML, CSS или браузерном JavaScript. Он должен храниться только как секрет серверной прослойки.

## 5. Основные функции сайта

### Главная страница

На главной странице студент заполняет кейс:

- название кейса;
- исходный язык;
- целевой язык;
- адресат;
- жанр / контекст;
- цель перевода;
- стратегия;
- исходный текст.

Данные сохраняются в `localStorage` браузера.

### Этап 1 — промпты A/B/C

Сайт автоматически формирует три варианта промпта:

- вариант A — ориентация на академическую аудиторию и форенизацию;
- вариант B — ориентация на массовую аудиторию и доместикацию;
- вариант C — перевод с культурологическим комментарием.

Промпты остаются редактируемыми. Студент может вручную исправить или дополнить промпт перед использованием во внешней LLM.

### Этап 2 — аудит генерации

Этап содержит интерактивный аудит-лист:

- фрагмент;
- тип риска;
- почему это риск;
- предлагаемая правка;
- отметка о внесении правки.

Можно добавлять и удалять строки. Экспорт CSV выполняется с UTF-8 BOM, чтобы файл корректно открывался в Microsoft Excel.

### Этап 3 — постредактирование и комментарий

Студент вводит финальный перевод, формирует переводческий комментарий и заполняет мини-глоссарий из 10 единиц.

Глоссарий включает:

- исходную культурно маркированную единицу;
- переводческое решение;
- тип компенсации.

Глоссарий включается в итоговый TXT-отчёт и передаётся в раздел оценивания.

### Оценивание

Оценивание проводится по трём критериям:

- **ПА** — прагматическая адекватность;
- **РЦИ** — распознавание цифровой интерференции;
- **ПК** — приёмы компенсации.

Каждый критерий оценивается по шкале 0–5. Максимальный итоговый балл — 15.

## 7. Хранение данных

Данные сохраняются локально в браузере через `localStorage`.

Сайт не отправляет данные на сервер в автономном режиме. Данные покидают устройство только если пользователь самостоятельно копирует текст во внешнюю LLM или включает API-режим через `assets/config.js`.

Кнопка очистки кейса требует подтверждения, чтобы избежать случайной потери работы.

## 8. Публикация на GitHub Pages

1. Загрузите все файлы проекта в репозиторий.
2. Откройте настройки репозитория.
3. Перейдите в `Settings → Pages`.
4. Выберите `Deploy from a branch`.
5. Укажите ветку `main` и папку `/(root)`.
6. Сохраните настройки.

После публикации сайт будет доступен по адресу GitHub Pages.

## 9. Проверка перед демонстрацией

Перед защитой или демонстрацией рекомендуется проверить:

1. открывается ли `index.html`;
2. сохраняется ли кейс;
3. генерируются ли промпты A/B/C;
4. копируются ли промпты;
5. работает ли добавление и удаление строк в аудит-листе;
6. скачивается ли CSV;
7. сохраняется ли финальный перевод;
8. заполняется ли глоссарий из 10 единиц;
9. скачивается ли итоговый TXT-отчёт;
10. работает ли расчёт ПА / РЦИ / ПК;
11. открываются ли файлы из раздела «Материалы»;
12. работает ли кнопка возврата на главную.

## 10. Методическое значение

Сайт демонстрирует не просто использование LLM, а управляемую методическую процедуру работы с машинной генерацией: постановка переводческой задачи, проектирование промпта, критическая проверка результата, выявление цифровой интерференции, применение приёмов компенсации и итоговая рефлексия.
